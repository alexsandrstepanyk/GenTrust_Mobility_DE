## Технічна діаграма компонентів

```
┌─────────────────────────────────────────────────────────────────┐
│                      QuestsScreen.tsx                           │
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  State Management                                         │ │
│  │  • availableQuests: Quest[]                               │ │
│  │  • activeQuest: Quest | null                              │ │
│  │  • modalVisible: boolean                                  │ │
│  │  • selectedQuest: Quest | null                            │ │
│  │  • modalLoading: boolean                                  │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │  Functions                                                │ │
│  │  • handleTakeQuest(questId)                               │ │
│  │     → Знаходить квест                                     │ │

│  │     → setModalVisible(true)                               │ │

│  │     → POST /api/quests/{id}/take                          │ │

│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │

│  │  • Список квестів (map)                                   │ │
```

## API Flow

```
MOBILE APP                    API SERVER                  DATABASE
    │                             │                           │
    │  1. GET /api/quests/       │                           │
    │     available              │                           │
    ├───────────────────────────>│                           │
    │                            │  SELECT * FROM quests     │
    │                            │  WHERE status='AVAILABLE' │
    │                            ├─────────────────────────>│
    │                            │                           │
    │                            │<──────────────────────────┤
    │  [25 quests]               │  Return 25 quests         │
    │<───────────────────────────┤                           │
    │                             │                           │
    │  2. User натискає          │                           │
    │     "Get order"            │                           │
    │                             │                           │
    │  3. Shows modal            │                           │
    │     (client-side)          │                           │
    │                             │                           │
    │  4. User підтверджує       │                           │
    │     (2 натискання)         │                           │
    │                             │                           │
    │  5. POST /api/quests/      │                           │
    │     {questId}/take         │                           │
    │     Headers: Bearer token  │                           │
    ├───────────────────────────>│                           │
    │                            │  Verify token             │
    │                            │  Check quest available    │
    │                            │  Check user has no active │
    │                            │                           │
    │                            │  UPDATE quests            │
    │                            │  SET status='IN_PROGRESS',│
    │                            │      userId=...           │
    │                            │  WHERE id=...             │
    │                            ├─────────────────────────>│
    │                            │                           │
    │                            │<──────────────────────────┤
    │  {success: true}           │  Quest updated            │
    │<───────────────────────────┤                           │
    │                             │                           │
    │  6. Close modal            │                           │
    │     Show Alert "Успіх!"    │                           │
    │                             │                           │
    │  7. GET /api/quests/       │                           │
    │     available (refresh)    │                           │
    ├───────────────────────────>│                           │
    │  [24 quests]               │  (один квест менше)       │
    │<───────────────────────────┤                           │
    │                             │                           │
    │  8. GET /api/users/        │                           │
    │     active-quest           │                           │
    ├───────────────────────────>│                           │
    │  {quest with status        │                           │
    │   IN_PROGRESS}             │                           │
    │<───────────────────────────┤                           │
    │                             │                           │
```
