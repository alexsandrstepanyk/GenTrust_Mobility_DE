# Release v6.0.7 - Dynamic Departments + Auto-Launch + Full Sync

**Дата:** 2026-04-07
**Версія:** v6.0.7
**Статус:** Dynamic Department Release Ready

## Що увійшло в реліз

- Додано створення нових департаментів із City-Hall Dashboard.
- Додано автоматичне створення окремої SQLite БД для кожного нового департаменту.
- Додано автоматичне застосування схеми департаменту до нової БД.
- Додано динамічний registry департаментів з автопризначенням порту.
- Додано автозапуск нових департаментів у full startup script.
- Додано автопояву нових департаментів на Monitor Dashboard `9000`.
- Додано перевірку БД динамічних департаментів у Monitor Dashboard.
- Виправлено Socket.IO в основному `api-server.ts`.
- Виправлено JWT-перевірку для Socket.IO.
- Додано realtime події `report:new`, `stats:update`, `user:registered`.
- Додано idempotency + transaction для `task_orders approve`.
- Додано idempotency + transaction для `quests complete`.
- Додано синхронізацію `Report` між головною БД та БД департаменту.

## Результат

Після цього релізу новий департамент створюється через UI, отримує власну БД, синхронізується з основною системою, з'являється в автозапуску і відображається в Monitor Dashboard без ручного додавання в код кожного сервісу.

## Відповідність правилам start.sh

- Код не видалявся, зміни вносилися розширенням існуючої логіки.
- Документацію оновлено в `README.md`, `ROADMAP.md` і `start.sh`.
- Порти залишено в діапазоні `3000-9000`.
- Для динамічних департаментів використовується стабільне автопризначення порту починаючи з `5188`.
