# GenTrust Mobility — інвесторський огляд

## 1) Короткий опис
GenTrust Mobility — екосистема Telegram‑ботів для міського інспектування інфраструктури та керування комунальними задачами. Скаути фіксують проблеми (яма, сміття, освітлення) через фото та геолокацію, а міські служби отримують підтверджені задачі для виконання. Додатково працює модуль логістичних квестів для мотивації та фінансового стимулу.

## 2) Цінність для міста та бізнесу
- **Оперативність**: проблеми фіксуються в реальному часі з координатами та фото.
- **Пріоритизація**: автоматичні категорії та статуси підвищують якість відбору.
- **Прозорість**: рейтинг та історія дій дозволяють аналізувати ефективність.
- **Масштабованість**: архітектура підтримує кілька ботів та ролей.

## 4) Основні ролі в системі
1. **Скаут** — надсилає звіти про проблеми, виконує квести.
2. **Адміністратор міста (City Hall)** — модерує звіти, керує пріоритетами.
3. **Комунальний працівник** — отримує підтверджені задачі та виконує їх.
4. **Провайдер (бізнес/школи/ГО)** — створює логістичні задачі.
5. **Master Core** — центральний контроль доступів і модерації.

## 5) Логіка роботи (енд‑ту‑енд)
### 5.1 Потік “Звіт про інфраструктуру”
1) Скаут запускає “Звіт” → надсилає фото.
2) Фото аналізує Gemini AI → пропонується категорія.
3) Скаут підтверджує категорію та надсилає геолокацію.
4) Звіт зберігається в БД зі статусом PENDING.
5) City Hall отримує повідомлення з кнопками “Схвалити/Відхилити”.
6) При схваленні скаут отримує рейтинг і формується задача для комунальної служби.

### 5.2 Потік “Комунальні служби”
1) Працівник реєструється через бот.
2) Заявка надходить на модерацію.
3) Після схвалення працівник бачить список задач.
4) Приймає задачу → виконує → закриває.

### 5.3 Потік “Логістичні квести”
1) Скаут переглядає доступні квести у своєму місті.
2) Бере квест → отримує коди отримання/доставки.
3) Після введення правильного коду квест завершується.

## 6) Модель даних (основні сутності)
- **User** — профіль скаута/адміна.
- **Report** — звіти про проблеми інфраструктури.
- **Quest** — логістичні задачі.
- **MunicipalWorker** — працівники комунальної служби.
- **MunicipalTask** — задачі для комунальних служб.
- **LeaderboardSnapshot** — історія рейтингів.

## 7) Технологічний стек
- **Node.js + TypeScript**
- **Telegraf** (Telegram Bot Framework)
- **Prisma + SQLite** (можливий перехід на PostgreSQL)
- **Gemini AI** (візуальний аналіз фото)

## 8) Безпека і контроль
- Ролі та статуси обмежують доступ.
- Всі дії логуються.
- Окремий Master Core для централізованого управління.

## 9) Масштабування і монетизація
- **Підписка для міст** (SaaS модель).
- **Аналітика для муніципалітетів**.
- **Партнерства з бізнесом** (квести/логістика).

## 10) Роадмап
- Перехід на PostgreSQL та cloud‑інфраструктуру.
- Мобільний додаток на базі цієї логіки.
- Розширена аналітика та звіти для міст.
