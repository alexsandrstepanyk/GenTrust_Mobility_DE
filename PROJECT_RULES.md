## ⚠️ ПРАВИЛО 4: ОБМЕЖЕННЯ ДІАПАЗОНУ ПОРТІВ (КРИТИЧНО!)

```
📍 Твій діапазон роботи: 3000 - 9000
❌ ЗАБОРОНЕНО чіпати порти ПОЗА цим діапазоном
❌ НЕ зупиняти процеси на портах < 3000 або > 9000
❌ НЕ перевіряти порти поза діапазоном (3000-9000)
```

**Чому це важливо:**
Це запобігає зупинці сторонніх сервісів:
- MySQL (3306) - ❌ НЕ ЧІПАТИ
- Redis (6379) - ❌ НЕ ЧІПАТИ
- MongoDB (27017) - ❌ НЕ ЧІПАТИ
- PostgreSQL (5432) - ❌ НЕ ЧІПАТИ

**Приклади:**

```bash
# ✅ ПРАВИЛЬНО: Чистимо тільки наші порти (3000-9000)
for port in 3000 5173 5174 5180 8082 8083 9000; do
    lsof -ti:$port | xargs kill -9
done

# ❌ НЕПРАВИЛЬНО: Не чистимо всі порти підряд
# killall -9 node  # МОЖЕ ВБИТИ MySQL, Redis, тощо
```

**Безпечні команди:**
```bash
# ✅ Безпечно: тільки наші порти
lsof -ti:3000 | xargs kill -9
lsof -ti:9000 | xargs kill -9
lsof -ti:5173 | xargs kill -9

# ❌ Небезпечно: може зачепити сторонні сервіси
# killall -9 node
# pkill -9 node
```
