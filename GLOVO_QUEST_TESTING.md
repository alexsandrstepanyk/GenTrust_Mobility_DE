# 🎯 Тестування Glovo-like Квеста (Задан ння Прийняття)

## Огляд
Тепер мобільний додаток має **професійний потік прийняття завдань** як у Glovo курієра:

1. **Попередження** - "Це все дуже серйозно!"
2. **Детальна карта** з локацією завдання  
3. **Повна інформація** про завдання
4. **Двохкрокове підтвердження** для безпеки

## Перед початком тестування

### Переконайтесь що запущено:

```bash
# 1. Бекенд API (порт 3000)
npm run api

# 2. Експо мобільний додаток (порт 8081)
cd mobile-school
npm start

# 3. Відкрити Expо на телефоні
```

### Інша потрібна інформація:

- **IP Мак**: 192.168.178.34
- **Expo Tunnel**: Активний і синхронізований з телефоном
- **Тестові дані**: 25 квестів в базі
- **Авторизація**: admin / admin

## Кроки Тестування

### 1️⃣ Вхід в додаток

```
1. Відкрити Expo на телефоні
2. Увійти: email=admin@test.com password=admin
3. Переходимо на вкладку "📦 Завдання" (Tasks)
```

Очікуваний результат: Виділяємо список доступних завдань (25 квестів)

---

### 2️⃣ Натискаємо "Отримати замовлення" (Get order)

```
1. На карточці будь-якого завдання натиснути кнопку "Get order"
2. Відкривається модаль з попередженням
```

**Очікуваний результат:**
- ⚠️ Велике вікно попередження
- Текст: "Серйозне попередження"
- Основний текст про відповідальність
- Два кнопки: "Повернутися" і "Прочитати детально"

---

### 3️⃣ Натискаємо "Прочитати детально"

```
1. Натиснути кнопку "Прочитати детально" (Read Details)
2. Модаль переходить до деталей завдання
```

**Очікуваний результат:**
Виділяються (по порядку):

- 📍 **Заголовок**: Назва завдання + Винагорода (₴XX)
- 📝 **Опис**: Повний текст завдання
- 🗺️ **Карта**: MapView з маркером локації  
- 📌 **Адреса**: Місто, Район, Точна локація
- ℹ️ **Деталі**: 
  - Тип завдання
  - Статус
  - Дата створення
- ✓ **Умови**:
  - Ви повинні завершити за вказаний час
  - Слідуйте інструкціям
  - Звітуйте про завершення
  - Якість впливає на рейтинг

---

### 4️⃣ Натискаємо "ПРИЙНЯТИ ЗАВДАННЯ"

```
1. Натиснути синю кнопку "ПРИЙНЯТИ ЗАВДАННЯ" (ACCEPT TASK)
2. Текст змінюється на "ПІДТВЕРДИТИ ПРИЙНЯТТЯ"
3. Натиснути знову
```

**Очікуваний результат:**
- Після першого натиску: кнопка змінює текст на "ПІДТВЕРДИТИ ПРИЙНЯТТЯ"
- Після другого натиску: Показується успішне повідомлення
- Модаль закривається
- Вертаємось до списку завдань
- Завдання **більше не відображається в списку** (прийнято)
- Вище з'явиться **"Активне завдання"** зеленим блоком

---

### 5️⃣ Перевіримо активне завдання

```
1. Перевіримо новий зелений блок "🎯 Active Quest"
2. Натиснути кнопку "Завершити" (Complete)
```

**Очікуваний результат:**
- Зелений блок показує прийняте завдання
- Кнопка "Complete" переходить на екран завершення

## Технічні Характеристики

### Стек:
- **Frontend**: React Native + Expo SDK 54
- **Maps**: react-native-maps
- **State Management**: useState hooks
- **Async**: axios + SecureStore для токена
- **Localization**: i18next з 3 мовами

### API Endpoints:

```
GET  /quests/available      - Отримати доступні завдання
GET  /users/active-quest    - Отримати активне завдання
POST /quests/{id}/take      - Прийняти завдання
```

### Локалізація:

Три мови:
- 🇺🇦 Українська (uk)
- 🇩🇪 Deutsch (de)  
- 🇬🇧 English (en)

Змінити мову можна в Settings → Language
