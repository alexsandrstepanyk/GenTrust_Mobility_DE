# ⚡ Шпаргалка - Швидкий відкат

## 🔄 Після КОЖНОЇ вашої правки

Я **автоматично** створюю:
- ✅ Commit з описом
- ✅ Тег з timestamp (напр. `v3.1.1-20260224-0130`)
- ✅ Локальний бекап

---

## 🚀 Швидкий відкат

### 1. Переглянути всі теги

```bash
git tag -l
```

**Приклад виводу:**
```
v3.1.1-20260224-0125
v3.1.1-20260224-0130
v3.1.1-20260224-0145
v3.1.2-20260224-0200
```

---

### 2. Відкат до конкретної версії

```bash
# Переглянути зміни в тегу
git show v3.1.1-20260224-0130

# Відкатитись (переглянути)
git checkout v3.1.1-20260224-0130

# Створити гілку з тегу
git checkout -b rollback-to-v3.1.1 v3.1.1-20260224-0130
```

---

### 3. Повний відкат змін

```bash
# Скасувати останню зміну
git reset --hard HEAD~1

# Відкат до конкретного тегу
git reset --hard v3.1.1-20260224-0130

# Примусовий push (якщо потрібно)
git push -f origin main
```

---

## 📊 Останні 5 тегів

```bash
git tag -l | tail -5
```

---

## 🔍 Порівняння версій

```bash
# Порівняти дві версії
git diff v3.1.1-20260224-0125 v3.1.1-20260224-0130

# Показати зміни в версії
git show v3.1.1-20260224-0130 --stat
```

---

## ⚡ Найшвидший відкат

**Якщо все зламалось після останньої зміни:**

```bash
# 1. Скасувати зміни
git reset --hard HEAD~1

# 2. Або відкат до попереднього тегу
git checkout $(git tag -l | tail -2 | head -1)
```

---

## 📋 Формати тегів

| Формат | Приклад | Опис |
|--------|---------|------|
| `v3.1.1-20260224-0130` | PATCH + timestamp | Кожна зміна |
| `v3.1.2` | MINOR | Середні зміни |
| `v4.0.0` | MAJOR | Глобальні зміни |

---

## 💡 Корисні команди

```bash
# Останній тег
git describe --tags --abbrev=0

# Всі теги з описом
git tag -l -n5

# Хто створив тег
git show v3.1.1-20260224-0130

# Видалити локальний тег
git tag -d v3.1.1-20260224-0130

# Видалити тег з GitHub
git push origin :refs/tags/v3.1.1-20260224-0130
```

---

## 🎯 Приклад швидкого відкату

**Сценарій:** Ви змінили файл, все зламалось.

**Рішення за 3 команди:**

```bash
# 1. Знайти останній стабільний тег
git tag -l | tail -5

# 2. Відкатитись
git checkout v3.1.1-20260224-0130

# 3. Перевірити
npm run dev  # або інша команда
```
