# 🔄 Швидкий відкат (Rollback)

## 🤖 Автоматична система бекапів

Після **КОЖНОЇ** вашої зміни я автоматично:

1. ✅ Створюю commit з описом
2. ✅ Створюю тег (напр. `v3.1.1`, `v3.1.2`)
3. ✅ Пушу на GitHub

## 🔄 Як відкатитись назад

### Варіант 1: Швидкий відкат до тегу

```bash
# Переглянути всі теги
git tag -l

# Відкатитись до конкретної версії
git checkout v3.1.2

# Або створити гілку з тегу
git checkout -b rollback-to-v3.1.2 v3.1.2
```

### Варіант 2: Відкат через GitHub

1. Відкрийте https://github.com/YOUR_USERNAME/GenTrust_Mobility_DE/tags
2. Знайдіть стабільну версію (напр. v3.1.2)
3. Натисніть на тег
4. Завантажте код або створіть гілку

### Варіант 3: Повний відкат змін

```bash
# Скасувати останню зміну (зберегти файли)
git reset --soft HEAD~1

# Скасувати останню зміну (видалити файли)
git reset --hard HEAD~1

# Відкат до конкретної версії
git reset --hard v3.1.2

# Push з примусом (обережно!)
git push -f origin main
```

## 📋 Теги та версії

### Формат тегів:

| Тег | Опис | Коли |
|-----|------|------|
| `v3.1.1` | PATCH | Виправлення багів |
| `v3.1.2` | MINOR | Нові функції |
| `v4.0.0` | MAJOR | Глобальні зміни |

### Перегляд тегів:

```bash
# Всі теги
git tag -l

# Теги з описом
git tag -l -n10

# Останні 5 тегів
git tag -l | tail -5

# Деталі тегу
git show v3.1.2
```

## 🚀 Швидкий відкат після невдалої зміни

```bash
# 1. Якщо зміни ще не закоммічені
git reset --hard HEAD

# 2. Якщо commit вже створено
git reset --hard HEAD~1

# 3. Якщо потрібно відкат до конкретної версії
git checkout v3.1.2

# 4. Якщо потрібно скасувати push
git push -f origin v3.1.2:main
```

## ⚡ Шпаргалка

| Команда | Що робить |
|---------|-----------|
| `git tag -l` | Показати всі теги |
| `git checkout v3.1.2` | Відкат до версії v3.1.2 |
| `git reset --hard HEAD~1` | Скасувати останню зміну |
| `git show v3.1.2` | Показати зміни в версії v3.1.2 |
| `git diff v3.1.2 v3.1.3` | Порівняти дві версії |

## 🎯 Приклад використання

**Сценарій:** Ви внесли зміни, але все зламалось.

**Рішення:**

```bash
# 1. Подивитись історію тегів
git tag -l

# 2. Відкатитись до стабільної версії
git checkout v3.1.2

# 3. Перевірити чи працює
# ... тестування ...

# 4. Якщо працює - створити гілку для фіксу
git checkout -b fix-from-v3.1.2

# 5. Виправити проблему
# ... правки ...

# 6. Закоммітити і запушити
git add -A
git commit -m "fix: Виправлено проблему"
git push origin fix-from-v3.1.2
```
